# Mission: Learn Laravel & Svelte from SecureCAT

## Who
David — BSIT student, lead developer of SecureCAT-v2. Has worked on the project but feels there's technical debt and gaps in foundational understanding of Laravel and Svelte that a junior developer would miss.

## Why
- Build genuine understanding of Laravel patterns (routes, controllers, services, Eloquent, policies, etc.) so the code you write is intentional, not cargo-culted
- Learn Svelte 5 fundamentals through real components this project already uses
- Spot technical debt and anti-patterns before they compound
- Become a more confident full-stack developer, not just someone who can make things "work"

## What we'll do
Walk the SecureCAT codebase systematically. Each session picks one concrete concept:

1. **Laravel request lifecycle** — how a URL becomes a response (routes -> controllers -> services -> views)
2. **Eloquent relationships** — reading the actual model relationships in SecureCAT
3. **Service layer pattern** — why SecureCAT has so many service classes
4. **Form Requests & validation** — how validation is extracted from controllers
5. **Svelte reactivity** — how SecureCAT components manage state
6. **Inertia bridging** — how Laravel and Svelte talk to each other
7. ...and more as we go

Each session produces:
- A **lesson** (reference HTML you can revisit)
- A **learning record** (captures the insight)
- Real code from SecureCAT as the example

## Stretch goals
- By the end, you should be able to look at any SecureCAT controller or Svelte component and explain what every line does and why it's structured that way
- You should be able to identify at least 3 refactoring opportunities yourself
